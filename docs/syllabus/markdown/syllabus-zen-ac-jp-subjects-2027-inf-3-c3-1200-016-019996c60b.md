## 本文

## 概要

プロジェクトマネジメント概論で学修した知識を用いて、初めて行うプロジェクトに適用し、実践を通じて学修する。演習形式で進め、プロジェクトの目標設定からニーズ把握、計画作成、スケジューリング、リスクと品質の検討、見積り、そして計画書作成までの一連のプロセスを体験する。作業の実施、進捗の確認、レビューを経て、最終的には自身が行ったプロジェクトについてプレゼンテーションを行う。これらの過程を通じて、プロジェクトマネジメントの各段階で必要な知識と技術を深め、実際の現場での応用力を鍛えることを目指す。

## 到達目標

プロジェクトマネジメント応用を通じて、プロジェクトの目的、目標の作成、ニーズ把握、WBSの作成、スケジューリング、リスクと品質の管理、レビューと試験、プレゼンテーションなどを体験することにより、新規のプロジェクトに対してもプロジェクトとして全体像をイメージし、適切に実施していく技術を身につけることができる。また、情報表現力とコミュニケーション能力も高まり、プロジェクトを実施することによる課題解決能力を身につけることを目標とする。

## 講義計画

| 回 | タイトル | 内容 |
| --- | --- | --- |
| 1 | オリエンテーション&プロジェクトマネジメントについて | 本授業で行うプロジェクトと進め方についてのオリエンテーションとプロジェクトマネジメントについての説明を行う。 |
| 2 | プロジェクト目標の作成 | プロジェクトで開発するものについて、背景や目的、目標を議論し、それをプロジェクトチャーターとしてまとめる。 |
| 3 | ニーズの把握 | 聞き取り調査を行い、将来の利用者のニーズを把握し、まとめる。 |
| 4 | プロジェクト計画の作成　（WBS） | 目標を達成するための作業を洗い出し、WBS（Work Breakdown Structure）を作成する。 |
| 5 | スケジューリング | WBSの作業をスケジュールに展開する。さらに相互に、そのスケジュールが適正であるかのチェックを行う。 |
| 6 | リスクの検討 | リスクの洗い出しを行い、分析・評価を実施する。そして、対応策を検討し、WBS、スケジュールの更新を行う。 |
| 7 | 品質確保の方法を検討 | フェーズごとに、どのような品質確認を行うかを検討し、確認可能な目標を設定する。また、必要に応じてWBSおよびスケジュールを更新する。 |
| 8 | 見積り | 各作業に必要となる工数、コストの見積もりを行い、EVMにおけるPV線を作成する。 |
| 9 | プロジェクト計画書の完成 | ここまでの検討をもとに、プロジェクト計画書を完成させる。 |
| 10 | プロジェクト実施 | プロジェクト計画書をもとに、実際に作業を実施する。この後、作業の実施は13回まで続く。 |
| 11 | 進捗の確認（１） | 現時点での進捗を集計し、EVM(Earned Value Management)を用いた進捗報告書を作成する。なお、作業を継続する。 |
| 12 | 進捗の確認（２） | 現時点での進捗を集計し、EVM(Earned Value Management)を用いた進捗報告書を作成する。なお、作業を継続する。 |
| 13 | レビューと試験 | 出来上がってきた成果物についてのレビューおよび試験を行う。 |
| 14 | プレゼンテーション | 完成した成果物と、プロジェクトの経過についてプレゼンテーションを実施する。 |
| 15 | 振り返り | プロジェクトを通じて、困難に感じたこと、うまくいったことを整理し、振り返りを行う。また、次回、同様の作業をする人向けの引継ぎ事項をまとめる。 |
